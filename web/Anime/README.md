# Anime Vault

一个面向个人使用的线上番剧观看管理网页。如果你所看的不同番剧在多个不同的网站或者平台，你可以用这个项目进行统一化管理。每个番剧的播放链接都是独立且由用户自定义配置的。项目中内置了bilibili某番剧的网址配置参数。

项目提供：

- 首页海报墙浏览
- 番剧详情页展示
- 本地可编辑并持久保存的番剧播放地址
- 可配置的剧集跳转规则
- 剧集总数、集数查询偏移、地址拼接参数管理
- 上一次播放集数记录

整个项目基于标准库实现，不依赖第三方 Python 包，适合本地快速部署和长期自用。

## 适用场景

- 想把常看的番剧整理成一个本地海报墙
- 想为每部番剧保存自己的播放入口
- 想按固定地址规则快速跳转各集
- 想记录最近看到哪一集

## 当前功能

### 1. 首页海报墙

- 从 `poster/` 目录读取海报资源
- 展示番剧卡片、标题、副标题和发行信息
- 支持即时搜索筛选

### 2. 番剧详情页

- 展示海报、横版剧照背景、简介、剧集演员、关键词、资料来源
- 支持本地编辑番剧播放地址
- 默认点击播放地址文本框会直接打开该地址
- 只有切换到“编辑地址”状态后才允许修改并保存

### 3. 剧集系统

- 在详情页底部展示“剧集”面板
- 点击“配置”后可设置：
  - 总集数
  - 根域名
  - 路由
  - 查询参数前缀
  - 集数查询偏移
  - 其他
- “集数”字段由系统自动生成，不需要手动输入
- 点击某一集后，系统会把地址按以下顺序拼接：

```text
根域名 + 路由 + 查询参数前缀 + 实际播放集数 + 其他
```

- 页面显示的剧集编号始终是 `1, 2, 3...`
- 真正拼接到地址中的“实际播放集数”会受“集数查询偏移”影响

示例：

```text
总集数 = 25
集数查询偏移 = 0
```

则：

- 第 1 集 -> 拼接值为 `0`
- 第 25 集 -> 拼接值为 `24`

### 4. 上一次播放记录

- 点击某一集后，数据库会记录最近一次播放的集数
- 详情页右侧会显示“上一次播放”
- 最近播放的那一集会在剧集列表中高亮

## 技术结构

- 后端：`Python 3` 标准库
- Web 服务：`http.server.ThreadingHTTPServer`
- 数据库：`SQLite`
- 前端：原生 `HTML + CSS + JavaScript`

## 代码结构

- `app.py`
  极薄的启动入口，方便继续用 `python3 app.py` 运行。
- `anime_vault/cli.py`
  命令行参数解析与服务启动。
- `anime_vault/server.py`
  HTTP 路由、请求处理和页面响应。
- `anime_vault/repository.py`
  SQLite 初始化、迁移和数据读写。
- `anime_vault/renderers.py`
  模板渲染、HTML 片段拼接和剧集地址生成。
- `anime_vault/seed.py`
  番剧基础种子数据。
- `anime_vault/config.py`
  项目路径配置。

## 运行方式

在项目根目录执行：

```bash
python3 app.py
```

默认访问地址：

```text
http://127.0.0.1:8000
```

只初始化数据库：

```bash
python3 app.py --init-db
```

自定义端口：

```bash
python3 app.py --port 8080
```

更完整的运行说明见 [docs/运行教程文档.md](docs/运行教程文档.md)。
