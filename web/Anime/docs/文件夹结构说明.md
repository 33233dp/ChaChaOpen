## 目录职责

- `app.py`
  极薄的启动入口，用于兼容 `python3 app.py`。
- `anime_vault/cli.py`
  命令行入口，负责参数解析、数据库初始化和启动 HTTP 服务。
- `anime_vault/server.py`
  HTTP 请求处理器，负责首页、详情页、播放地址保存、剧集配置保存和剧集跳转。
- `anime_vault/repository.py`
  数据访问层，负责 SQLite 建表、字段迁移、查询和更新。
- `anime_vault/renderers.py`
  视图渲染层，负责模板加载、HTML 片段拼接和剧集地址生成。
- `anime_vault/seed.py`
  番剧基础种子数据。
- `anime_vault/config.py`
  项目路径配置。
- `README.md`
  项目总说明，包含功能概览、运行方式、目录说明和使用声明。
- `data/anime.db`
  SQLite 数据库，存放番剧标题、简介、演员、关键词、海报路径、剧照路径、播放地址、剧集配置和上一次播放记录。
- `poster/`
  首页海报墙使用的竖版海报资源。
- `Stills/`
  详情页使用的横版背景剧照资源。
- `templates/`
  页面模板，包含首页、详情页和新增番剧页 HTML。
- `static/`
  样式与前端交互脚本，负责海报墙视觉风格、详情页布局、搜索筛选、播放地址编辑模式和剧集配置展开逻辑。
- `docs/`
  项目结构、流程、运行、移植和资料来源说明。
- `prompt.md`
  初始任务要求记录文件。
