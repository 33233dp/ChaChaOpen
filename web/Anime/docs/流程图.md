# 文本流程图

## 1. 首页海报墙加载流程

```text
浏览器访问 /
  -> app.py / AnimeRequestHandler.do_GET()
  -> AnimeRequestHandler.render_home()
  -> load_catalog()
  -> SQLite: data/anime.db / anime 表
  -> render_card()
  -> templates/index.html
  -> static/styles.css + static/app.js
  -> 浏览器展示海报墙
```

## 2. 点击海报进入详情页流程

```text
用户点击海报卡片
  -> /anime/<slug>
  -> app.py / AnimeRequestHandler.do_GET()
  -> AnimeRequestHandler.render_detail(slug)
  -> get_anime(slug)
  -> load_catalog()
  -> templates/detail.html
  -> render_chip_list()
  -> render_source_list()
  -> render_playback_section()
  -> render_episode_section()
  -> 浏览器展示详情页
```

## 3. 数据初始化流程

```text
执行 python3 app.py
  -> main()
  -> ensure_database()
  -> 创建 data/anime.db 和 anime 表
  -> 补齐数据库缺失字段
  -> 将 CATALOG_SEED 写入数据库
  -> ThreadingHTTPServer 启动
```

## 4. 首页即时搜索流程

```text
用户输入搜索词
  -> static/app.js / updateFilter()
  -> 遍历 .poster-card 的 data-search
  -> 更新每张卡片 hidden 状态
  -> 更新 #visibleCount
  -> 无结果时显示 #emptyState
```

## 5. 保存番剧播放地址流程

```text
用户点击“编辑地址”
  -> static/app.js / syncEditingState()
  -> 文本框进入可编辑状态
用户修改后点击“保存地址”
  -> POST /anime/<slug>/playback-url
  -> app.py / AnimeRequestHandler.update_playback_url(slug)
  -> SQLite: anime.playback_url
  -> 302 跳回 /anime/<slug>
```

## 6. 保存剧集配置流程

```text
用户点击“配置”
  -> static/app.js / syncConfigState()
  -> 展开 .episodes-config
用户填写总集数、根域名、路由、查询参数前缀、起始集数、其他
  -> POST /anime/<slug>/episodes/config
  -> app.py / AnimeRequestHandler.update_episode_config(slug)
  -> SQLite: anime.episode_count
  -> SQLite: anime.episode_root_domain
  -> SQLite: anime.episode_route
  -> SQLite: anime.episode_query_prefix
  -> SQLite: anime.episode_start_number
  -> SQLite: anime.episode_other
  -> 302 跳回 /anime/<slug>
```

## 7. 点击剧集播放流程

```text
用户点击“第 N 集”
  -> GET /anime/<slug>/episode/<n>
  -> app.py / AnimeRequestHandler.play_episode(slug, n)
  -> compose_episode_url()
  -> 读取起始集数并计算真实播放编号
  -> SQLite: anime.last_played_episode
  -> 302 跳转到拼接后的播放地址
```
