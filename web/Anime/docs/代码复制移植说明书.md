# 代码复制移植说明书

## 最小复制清单

将以下文件和目录整体复制到目标机器或目标项目目录：

- `app.py`
- `README.md`
- `templates/`
- `static/`
- `poster/`
- `Stills/`
- `data/anime.db`
- `docs/`

## 移植步骤

1. 确认目标机器安装 `Python 3.10+`。
2. 将项目目录完整复制到目标位置。
3. 在目标目录执行 `python3 app.py`。
4. 打开浏览器访问 `http://127.0.0.1:8000`。

## 如果需要替换番剧资源

1. 将新海报放入 `poster/`。
2. 将新剧照放入 `Stills/`。
3. 修改 `app.py` 中的 `CATALOG_SEED`。
4. 执行 `python3 app.py --init-db` 重建数据库内容。

## 如果需要保留现有播放地址和剧集配置

- 直接复制现有的 `data/anime.db`
- 不要手动删除数据库文件
- `python3 app.py --init-db` 会补齐缺失字段，但不会覆盖用户已保存的播放地址和剧集配置

## 如果需要改端口

执行：

```bash
python3 app.py --port 9000
```

## 注意事项

- 图片文件名允许中文和空格，后端已经做 URL 编码处理。
- 页面依赖的全部逻辑均为本地文件，不依赖第三方 Python 包。
- 若你手动删除 `data/anime.db`，再次运行 `python3 app.py` 会自动重新生成数据库。
- 剧集地址规则、起始集数和上一次播放记录都保存在 `data/anime.db` 中。
- 项目源码本身不提供任何视频播放网址，播放地址需要由用户自行填写。
