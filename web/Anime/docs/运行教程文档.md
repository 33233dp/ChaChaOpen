# 运行教程文档

## 环境要求

- 操作系统：Windows 或 Ubuntu
- 浏览器：Google Chrome
- Python：3.10 及以上

## 启动步骤

在项目根目录执行：

```bash
python3 app.py
```

默认启动后会监听：

```text
http://127.0.0.1:8000
```

## 首次运行会发生什么

- 自动创建 `data/anime.db`
- 自动写入番剧基础数据
- 自动加载 `poster/` 和 `Stills/` 中的图片资源

## 指定端口启动

```bash
python3 app.py --port 8080
```

## 仅初始化数据库

```bash
python3 app.py --init-db
```

## 页面使用说明

- 首页展示番剧海报墙
- 点击任意海报可进入详情页
- 首页右上角搜索框可即时筛选番剧
- 详情页展示简介、剧集演员、关键词和资料来源
- 详情页支持本地保存番剧播放地址
- 详情页底部支持剧集配置和集数跳转

## 剧集配置说明

点击详情页底部的 `配置` 后，可以填写：

- `总集数`
  控制要生成多少个剧集按钮。
- `根域名`
  例如 `https://player.example.com`
- `路由`
  例如 `/watch`
- `查询参数前缀`
  例如 `?id=`
- `起始集数`
  用来控制真正拼接到 URL 里的起始编号。
- `其他`
  用来补充额外参数，例如 `&source=anime`

## 剧集地址拼接规则

最终播放地址按如下顺序拼接：

```text
根域名 + 路由 + 查询参数前缀 + 实际播放编号 + 其他
```

说明：

- 页面显示的集数始终是 `第 1 集`、`第 2 集`
- `实际播放编号` 会受到 `起始集数` 影响

示例：

```text
总集数 = 25
起始集数 = 0
查询参数前缀 = ?id=
```

则：

- 点击 `第 1 集` 会打开 `...?id=0`
- 点击 `第 25 集` 会打开 `...?id=24`

## 播放地址编辑说明

- 默认点击播放地址文本框时会直接打开当前地址
- 只有点击 `编辑地址` 后，文本框才允许修改
- 修改完成后点击 `保存地址` 即可写入本地数据库

## 播放记录说明

- 点击任意剧集后，会记录最近一次播放的集数
- 详情页右侧会显示 `上一次播放`
- 最近播放的那一集会高亮显示

## 常见问题

### 1. 浏览器无法打开页面

- 检查终端是否显示 `Serving on http://127.0.0.1:8000`
- 检查端口是否被其他程序占用

### 2. 图片不显示

- 确认 `poster/` 和 `Stills/` 目录仍然存在
- 确认图片文件名没有被改动

### 3. 想更新番剧内容

- 修改 `app.py` 中的 `CATALOG_SEED`
- 运行 `python3 app.py --init-db`
- 重新启动网页服务

### 4. 点了剧集没有正常跳转

- 检查是否已经填写 `根域名`
- 检查 `路由`、`查询参数前缀`、`其他` 是否拼接正确
- 检查 `起始集数` 是否符合目标站点的编号规则

### 5. 保存的地址或剧集配置没有生效

- 检查浏览器是否成功返回详情页
- 确认 `data/anime.db` 没有被删除
- 如你手动改过数据库结构，可以重新执行 `python3 app.py --init-db`
