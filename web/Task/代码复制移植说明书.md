# 代码复制移植说明书

## 目标

把这个任务管理应用复制到另一台 Windows 或 Linux 电脑后，仍然可以直接启动和使用。

## 需要复制的文件

- `app.py`
- `index.html`
- `heatmap.html`
- `运行教程文档.md`
- `代码复制移植说明书.md`
- `流程图说明.md`
- `项目结构说明.md`
- `webProgramming.md`

## 可选复制的文件

- `task_manager.db`

如果你想保留原来的任务数据，就把数据库文件一起复制过去。如果不复制，程序启动时会自动创建一个新的空数据库。

## 复制步骤

1. 把整个项目目录复制到目标机器。
2. 确认目标机器安装了 Python 3.10 或更高版本。
3. 在项目根目录打开终端。
4. 直接执行 `python app.py`。
5. 浏览器访问 `http://127.0.0.1:8000/`。
6. 如果要看热力图，访问 `http://127.0.0.1:8000/heatmap`。

如果目标机器没有 `python` 命令，就把第 4 步替换成 `python3 app.py`。

## 数据兼容性说明

- 数据库存储使用 SQLite，单文件形式，适合跨平台搬运。
- 任务字段采用通用文本和数字类型，不依赖特定平台。
- 如果目标环境中已有同名 `task_manager.db`，程序会直接复用其中的数据。

## 注意事项

- 不要修改 `index.html` 和 `app.py` 的相对路径关系，否则后端读取首页文件时会失败。
- 如果需要改端口，可以设置环境变量 `PORT` 后再启动，例如 `PORT=8080 python app.py`。
- 如果需要改端口，但系统只提供 `python3`，就使用 `PORT=8080 python3 app.py`。
