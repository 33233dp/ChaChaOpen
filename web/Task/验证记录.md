# 验证记录

## 已核验内容

- `app.py` 语法可由 `python -m py_compile` 检查。
- 后端会自动创建 SQLite 数据库和 `tasks` 表。
- 后端会自动创建 `task_events` 表，并通过 `/api/activity` 提供本年任务操作热力图数据。
- `index.html` 是任务页，`heatmap.html` 是独立热力图页，两者通过后端不同路由单独访问。
- 两个页面都通过同源 `fetch()` 调用后端接口，不需要额外跨域配置。
- 页面采用响应式布局，窄屏下侧边栏会自动堆叠到内容上方。
- 任务编辑、完成切换、删除和筛选都走统一接口，便于维护。
- 新增的年热力图会随创建、完成切换、编辑和删除操作自动刷新，并在独立页面中展示。

## 关注点

- 数据库文件是运行时生成的，不会在仓库初始化时预先存在。
- 页面字体优先使用系统可用字体，因此在不同系统上外观会略有差异，但布局保持一致。
