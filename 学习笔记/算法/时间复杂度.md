# 时间复杂度

## 1. 什么是时间复杂度？

**时间复杂度** 是衡量算法执行时间随输入数据规模增长而变化的量度。它描述的是算法运行时间与输入规模 `n` 之间的关系，而不是具体的执行时间（秒/毫秒）。

简单来说，时间复杂度回答的问题是：**当输入规模 n 增大时，算法的运行时间会如何增长？**

### 什么是输入规模（n）？

**输入规模** 指的是算法需要处理的数据量大小，通常用变量 `n` 表示。不同的算法问题，输入规模的含义不同：

| 算法问题 | 输入规模 n 的含义 |
|---------|-----------------|
| 数组求和 | n = 数组的元素个数 |
| 排序算法 | n = 待排序的元素个数 |
| 二分查找 | n = 有序数组的长度 |
| 矩阵乘法 | n = 矩阵的边长（如 n×n 矩阵） |
| 字符串匹配 | n = 字符串的长度 |
| 图算法 | n = 顶点数 / 边数 |
| 两数之和（LeetCode 1） | n = 数组中元素的个数 |

> **为什么关注输入规模？** 因为算法的运行时间几乎总是与输入规模相关。数组越长，求和越慢；字符串越长，匹配越慢。时间复杂度就是为了描述这种"数据量变大，时间如何变化"的规律。

例如，在 C++ 中：
```cpp
vector<int> arr1 = {1, 2, 3};                // n = 3
vector<int> arr2 = {1, 2, 3, ..., 1000000};  // n = 1,000,000
// 对 arr1 求和很快，但对 arr2 求和需要遍历 100 万个元素，耗时更长
// 这就是输入规模 n 对运行时间的影响
```

## 2. 大 O 表示法（Big O Notation）

大 O 表示法描述的是算法执行时间的**上界**（最坏情况），它关注的是**增长趋势**，而不是精确的执行次数。

大 O 表示法的简化原则：
- **忽略常数项**：O(2n) = O(n)，O(100n) = O(n)
- **忽略低阶项**：O(n² + n) = O(n²)，O(n + log n) = O(n)

## 3. 常见时间复杂度对比（效率从高到低）

| 时间复杂度 | 名称 | 说明 | 示例 |
|-----------|------|------|------|
| O(1) | 常数阶 | 执行时间与 n 无关 | 数组下标访问 |
| O(log n) | 对数阶 | 每步操作将问题规模减半 | 二分查找 |
| O(n) | 线性阶 | 时间与 n 成正比 | 线性查找 |
| O(n log n) | 线性对数阶 | 常见高效排序 | 归并排序、快速排序 |
| O(n²) | 平方阶 | 双层循环 | 冒泡排序 |
| O(2ⁿ) | 指数阶 | 增长极快，n≥30 时不可用 | 递归斐波那契 |
| O(n!) | 阶乘阶 | 最慢的增长速度 | 旅行商暴力解法 |

### 增长速度对比

假设计算机每秒执行 10⁸ 次操作，不同复杂度在 n 增大时的表现：

| n | O(log n) | O(n) | O(n log n) | O(n²) | O(2ⁿ) |
|---|---------|------|------------|-------|-------|
| 10 | < 1秒 | < 1秒 | < 1秒 | < 1秒 | < 1秒 |
| 100 | < 1秒 | < 1秒 | < 1秒 | < 1秒 | 4×10¹⁴年 |
| 10⁴ | < 1秒 | < 1秒 | < 1秒 | 1秒 | - |
| 10⁶ | < 1秒 | 0.01秒 | 0.2秒 | 2.8小时 | - |
| 10⁸ | < 1秒 | 1秒 | 27秒 | 317年 | - |

## 4. C++ 代码示例详解

### O(1) — 常数阶

执行时间恒定，与输入规模无关。

```cpp
int getFirst(const vector<int>& arr) {
    return arr[0];  // 只执行一次操作，无论数组多大
}

int sum(int a, int b) {
    return a + b;   // 固定操作
}
```

### O(log n) — 对数阶

每次操作将问题规模减半（二分法）。

**对数阶是如何出现的？**

对数阶（logarithmic time）出现的关键在于：**每一步操作都能将问题的规模缩小到原来的一个固定比例（通常是 1/2）**。这意味着随着问题规模的增大，所需操作的步数增长得非常缓慢。

> 💡 **直观理解**：如果你有一本 1024 页的书，要找到某一页，每次翻到中间位置——翻 1 次剩下 512 页，翻 2 次剩下 256 页，翻 3 次剩下 128 页……翻 10 次就找到了。这就是 log₂(1024) = 10。

**数学推导：**

假设初始规模为 `n`，每次操作后规模变为原来的 `1/2`，经过 `k` 次操作后规模变为 1：
- 第 1 次后：n/2
- 第 2 次后：n/4
- 第 3 次后：n/8
- ...
- 第 k 次后：n / 2ᵏ = 1

解方程 `n / 2ᵏ = 1` → `2ᵏ = n` → **k = log₂n**

所以需要操作的次数就是 log₂n，即 O(log n)。

**哪些场景会产生 O(log n)？**
- **二分查找**：在有序数组中查找目标值
- **二叉搜索树（BST）的查找/插入**：每次比较排除一半子树
- **平衡二叉树（如 AVL、红黑树）的操作**：树高为 log n
- **分治算法中只有一条递归分支**：每次递归只处理一半数据

```cpp
int binarySearch(const vector<int>& arr, int target) {
    int left = 0, right = arr.size() - 1;
    while (left <= right) {                      // 每次循环，搜索范围减半
        int mid = left + (right - left) / 2;
        if (arr[mid] == target) return mid;
        if (arr[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}
// 在 100 万个有序元素中查找，最多约 log₂(10⁶) ≈ 20 次比较
// 而 O(n) 的线性查找在最坏情况下需要 100 万次比较
```

> **为什么 log 的底数不重要？** 因为不同底数的对数之间只差一个常数倍：log₂n = log₃n × log₂3，常数倍在大 O 表示法中被忽略。

### O(n) — 线性阶

时间与输入规模成正比。

```cpp
int sumArray(const vector<int>& arr) {
    int sum = 0;
    for (int i = 0; i < arr.size(); i++) {   // n 次操作
        sum += arr[i];
    }
    return sum;
}
// n=100 时 100 次操作，n=10000 时 10000 次操作
```

```cpp
// 虽然是两次循环，但仍是 O(n)
void process(const vector<int>& arr) {
    for (int i = 0; i < arr.size(); i++) arr[i] *= 2;   // O(n)
    for (int i = 0; i < arr.size(); i++) arr[i] += 1;   // O(n)
    // 总：O(2n) = O(n)，常数项被忽略
}
```

### O(n log n) — 线性对数阶

许多高效排序算法的时间复杂度。

```cpp
// 归并排序
void merge(vector<int>& arr, int left, int mid, int right) {
    vector<int> temp(right - left + 1);
    int i = left, j = mid + 1, k = 0;
    while (i <= mid && j <= right) temp[k++] = (arr[i] <= arr[j]) ? arr[i++] : arr[j++];
    while (i <= mid) temp[k++] = arr[i++];
    while (j <= right) temp[k++] = arr[j++];
    for (int p = 0; p < k; p++) arr[left + p] = temp[p];
}

void mergeSort(vector<int>& arr, int left, int right) {
    if (left >= right) return;
    int mid = left + (right - left) / 2;
    mergeSort(arr, left, mid);        // 递归分解：log n 层
    mergeSort(arr, mid + 1, right);   // 递归分解：log n 层
    merge(arr, left, mid, right);     // 合并：每层 O(n)
}
// 总复杂度：O(n log n)
```

### O(n²) — 平方阶

常见于嵌套循环。n=1000 时就需要百万次操作。

```cpp
// 冒泡排序
void bubbleSort(vector<int>& arr) {
    int n = arr.size();
    for (int i = 0; i < n - 1; i++) {           // n 次
        for (int j = 0; j < n - 1 - i; j++) {   // n-i-1 次 ≈ n 次
            if (arr[j] > arr[j + 1]) {
                swap(arr[j], arr[j + 1]);
            }
        }
    }
}
// 总比较次数：n(n-1)/2 ≈ n²/2，即 O(n²)
```

```cpp
// 二维矩阵乘法
vector<vector<int>> matrixMultiply(
    const vector<vector<int>>& A,
    const vector<vector<int>>& B
) {
    int n = A.size();
    vector<vector<int>> C(n, vector<int>(n, 0));
    for (int i = 0; i < n; i++)          // n 次
        for (int j = 0; j < n; j++)      // n 次
            for (int k = 0; k < n; k++)  // n 次
                C[i][j] += A[i][k] * B[k][j];
    return C;
}
// 三重循环 → O(n³)
```

### O(2ⁿ) — 指数阶

增长非常快，n 稍大即不可用。

```cpp
// 斐波那契数列的递归实现（最坏情况）
int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
    // 每次调用产生 2 个子调用，深度为 n
    // 调用次数 ≈ 2ⁿ
}

// ✅ 优化版：O(n) 的迭代实现
int fibOptimized(int n) {
    if (n <= 1) return n;
    int a = 0, b = 1;
    for (int i = 2; i <= n; i++) {
        int temp = a + b;
        a = b;
        b = temp;
    }
    return b;
}
```

## 5. 如何分析时间复杂度？

### 步骤示例

```cpp
int example(vector<int>& arr) {
    // 步骤 1：找出基本操作（循环中的操作）
    int n = arr.size();
    int sum = 0;

    // O(n)
    for (int i = 0; i < n; i++) {
        sum += arr[i];
    }

    // O(n²)
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++) {
            sum += arr[i] * arr[j];
        }
    }

    return sum;
}
// 总复杂度 = O(n) + O(n²) = O(n²)
// 取增长最快的项，忽略低阶项
```

### 基本原则

1. **单层循环** → 看循环次数，通常是 O(n)
2. **嵌套循环** → 相乘，如双层循环 → O(n²)
3. **递归** → 看递归树的分支和深度
4. **取增长最快的项** → O(n² + n) = O(n²)
5. **舍去常数** → O(2n) = O(n)

## 6. 为什么重要？

- **面试必考**：大厂面试算法题的核心考察点之一
- **性能预估**：选择合适算法，避免 n 稍大就性能崩溃
- **架构设计**：高并发场景下，好的时间复杂度能节省大量服务器资源

> 选择算法时，先分析数据规模 n，再根据 n 的大小选择合适的时间复杂度。例如 n ≤ 20 时 O(2ⁿ) 也能接受，但 n = 10⁵ 时必须 O(n log n) 或更优。
