在实际的工程当中，为了减少功能包互相之间的依赖，通常会将接口定义在一个**独立的功能包**中，所以可以新建一个叫做`test_interfaces`的功能包，并把所有的接口都定义在这个独立的功能包里。
	- 新建msg文件夹，并在文件夹下新建`xxx.msg`
	- 在xxx.msg下编写消息内容并保存
	- 在CmakeLists.txt添加依赖和msg文件目录
	- 在package.xml中添加xxx.msg所需的依赖
	- 编译功能包即可生成python与c++头文件

**1 创建功能包，自定义接口**
在ws的src文件夹下面创建功能包
```
ros2 pkg create user_interface --build-type ament_cmake
```
**2 进入功能包user_interface文件夹新建msg文件夹和文件user.msg**
```
cd user_interface && mkdir msg && vim user.msg
```
**3 编写user.msg内容**
```
# 使用ros2的原始数据类型
string user_msg
# 使用标准接口std_msgs下的string类型
std_msgs/String content
# 图像消息，使用sensor_msgs的Image类型
sensor_msgs/Image image
```
**4 修改CMakeLists.txt**
``` 
# 添加sensor_msgs和std_msgs的功能包

find_package(ament_cmake REQUIRED)
find_package(std_msgs REQUIRED)
find_package(sensor_msgs REQUIRED)

# 添加自定义接口的依赖
find_package(rosidl_default_generators REQUIRED)

#添加消息文件和依赖
rosidl_generate_interface(${PROJECT_NAME}
	"msg/use.msg" #要和msg文件夹下的.msg文件名一致
	DEPENDENCIES sensor_msgs std_msgs
)

```
**5 修改package.xml**
```
<depend>sensor_msgs</depend>
<build_depend>rosidl_default</build_depend>
<exec_depend>rosidl_default</exec_depend>
<member_of_group>rosidl_interface_packages</member_of_group>
```
**6 编译**
```
colcon build --packages-select user_interfaces
```
