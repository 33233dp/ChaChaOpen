话题是没有返回的，适用于单向或大量的数据传递。而服务是双向的，客户端发送请求，服务端响应请求。
	- 同一个服务（名称相同）有且只能有一个节点来提供
	- 同一个服务可以被多个客户端调用

**内置服务**
装ROS2的时候其实系统为我们安装了一些样例程序，其中就有服务使用样例。
**启动服务端**
```
ros2 run examples_rclpy_minimal_service service
```
**使用命令查看服务列表**
```
ros2 service list
```
手动调用服务：客户端请求两个数字5+10，服务端返回15。
```
ros2 service call /add_two_ints example_interfaces/srv/AddTwoInts "{a: 5,b: 10}"
```
**常用服务命令**
	查看服务列表：ros2 service list
	手动调用服务：ros2 service call /add_two_ints example_interfaces/srv/AddTwoInts "{a: 5,b: 10}"
	查看服务接口类型：ros2 service find example_interfaces/srv/AddTwoInts
