# 栈（Stack）

栈是一种**线性数据结构**，它的特点是：

- **只允许在一端进行插入和删除**
- 这一端叫做**栈顶（top）**
- 另一端叫做**栈底**
- 满足 **后进先出（LIFO, Last In First Out）**

可以把它理解成一摞盘子：

- 新放上去的盘子在最上面
- 只能先拿最上面的盘子
- 最后放进去的，最先拿出来

---

## 1. 栈的逻辑结构

栈在逻辑上属于**线性表**的一种特殊形式。

普通线性表可以在中间任意位置插入、删除，而栈被限制为：

- 只能在栈顶 `push`（入栈）
- 只能在栈顶 `pop`（出栈）

例如：

```text
依次入栈: 1, 2, 3

栈顶
 ↓
[3]
[2]
[1]
 ↑
栈底
```

这时出栈顺序一定是：

```text
3 -> 2 -> 1
```

---

## 2. 栈的基本操作

栈最常见的操作有：

1. **初始化**
   创建一个空栈
2. **入栈（push）**
   把一个元素放到栈顶
3. **出栈（pop）**
   删除并返回栈顶元素
4. **取栈顶元素（top / peek）**
   只查看栈顶元素，不删除
5. **判空（empty）**
   判断栈中是否没有元素
6. **求大小（size）**
   返回当前元素个数

---

## 3. 栈的存储结构

栈的逻辑结构固定，但存储实现通常有两种：

## 3.1 顺序栈

顺序栈通常用**数组**实现。

需要两个核心信息：

- 一段连续存储空间
- 一个指针或下标 `top`，表示当前栈顶位置

示意：

```text
数组下标: 0   1   2   3   4
         [1] [2] [3] [ ] [ ]
                   ↑
                  top
```

特点：

- 访问栈顶元素很快
- 入栈、出栈时间复杂度通常是 `O(1)`
- 空间连续，缓存友好
- 但数组容量可能有限，可能会发生“栈满”

### 顺序栈入栈过程

1. `top` 向上移动
2. 将新元素放到 `top` 对应位置

### 顺序栈出栈过程

1. 取出 `top` 位置元素
2. `top` 向下移动

---

## 3.2 链式栈

链式栈通常用**链表**实现。

一般把**链表头部**作为栈顶，这样插入和删除更方便。

示意：

```text
top -> [3|next] -> [2|next] -> [1|null]
```

特点：

- 入栈、出栈也通常是 `O(1)`
- 不需要一块连续空间
- 理论上只要内存够，就可以不断扩展
- 但每个结点需要额外存储指针，空间开销更大

---

## 4. 栈的特点总结

- 栈是一种**受限线性表**
- 只允许在**栈顶**操作
- 遵循 **LIFO（后进先出）**
- 常见实现有：
  - **顺序栈：数组实现**
  - **链式栈：链表实现**

---

## 5. 时间复杂度

在常见实现下：

- `push`：`O(1)`
- `pop`：`O(1)`
- `top`：`O(1)`
- `empty`：`O(1)`

如果顺序栈发生扩容，那么单次扩容可能不是 `O(1)`，但很多语言的动态数组实现中，**均摊复杂度**仍可认为接近 `O(1)`。

---

## 6. 栈的典型应用

栈非常常见，主要用于“**最近进入的内容最先处理**”的场景。

### 6.1 函数调用栈

程序调用函数时，系统会把：

- 返回地址
- 局部变量
- 参数信息

压入调用栈中。函数结束后再弹出。

### 6.2 括号匹配

例如判断：

```text
() [] {}
```

是否配对正确。遇到左括号入栈，遇到右括号就和栈顶匹配。

### 6.3 表达式求值

例如中缀表达式、后缀表达式计算，经常借助栈完成。

### 6.4 深度优先搜索（DFS）

DFS 的递归本质上也依赖栈；非递归写法也常常显式使用栈。

### 6.5 撤销 / 回退

例如：

- 编辑器撤销操作
- 浏览器回退
- 软件历史记录

---

## 7. 栈和队列的区别

栈和队列都属于线性结构，但规则不同：

- **栈：后进先出（LIFO）**
- **队列：先进先出（FIFO）**

因此：

- 栈像一摞盘子
- 队列像排队买东西

---

## 8. 一句话理解栈

栈就是一种**只能在一端操作，并且后放进去的元素先出来**的数据结构。

---

## 9. C++ 中的栈

C++ STL 中有 `stack` 容器适配器，常用操作有：

```cpp
stack<int> s;
s.push(10);
s.push(20);
s.top();   // 20
s.pop();   // 弹出 20
s.empty(); // 是否为空
```

`stack` 默认底层通常基于 `deque` 实现，也可以用 `vector` 或 `list` 作为底层容器。

---

## 10. 调用栈（Call Stack）

调用栈是程序运行时非常重要的一个概念。

它本质上也是一个**栈结构**，遵循：

- **后调用的函数先返回**
- **先调用的函数后返回**

这正好符合栈的 **LIFO** 特性。

---

## 11. 什么是调用栈

当一个函数被调用时，系统通常会为这个函数分配一块栈上的空间，用来保存这次调用的信息。

这些信息组成一个**栈帧（stack frame）**，也叫：

- 活动记录
- 调用帧

每调用一次函数，就会把一个新的栈帧压入调用栈中；函数执行结束后，对应的栈帧再弹出。

因此可以理解为：

```text
函数调用 -> 入栈
函数返回 -> 出栈
```

---

## 12. 栈帧里通常有什么

一个函数对应的栈帧里，通常会保存以下信息：

- **函数参数**
- **局部变量**
- **返回地址**
- **保存的寄存器现场**
- 某些编译器生成的临时数据

也就是说，函数运行所需的“现场信息”通常都放在调用栈里。

这样函数执行结束后，程序才能准确回到原来的位置继续执行。

---

## 13. 调用栈的执行过程

看一个简单例子：

```cpp
void c() {
}

void b() {
    c();
}

void a() {
    b();
}

int main() {
    a();
}
```

执行过程大致是：

1. `main()` 开始执行，`main` 的栈帧入栈
2. `main()` 调用 `a()`，`a` 的栈帧入栈
3. `a()` 调用 `b()`，`b` 的栈帧入栈
4. `b()` 调用 `c()`，`c` 的栈帧入栈
5. `c()` 执行结束，`c` 的栈帧出栈
6. `b()` 执行结束，`b` 的栈帧出栈
7. `a()` 执行结束，`a` 的栈帧出栈
8. `main()` 执行结束，`main` 的栈帧出栈

调用栈变化示意：

```text
开始时：
[main]

main 调用 a：
[a]
[main]

a 调用 b：
[b]
[a]
[main]

b 调用 c：
[c]
[b]
[a]
[main]
```

返回时则反过来一层层弹出。

---

## 14. 调用栈和递归

递归本质上就是函数调用自己，而每调用一次，都会新建一个栈帧压入调用栈。

例如：

```cpp
int factorial(int n) {
    if (n == 1) return 1;
    return n * factorial(n - 1);
}
```

调用 `factorial(4)` 时，大致会形成：

```text
factorial(4)
factorial(3)
factorial(2)
factorial(1)
```

调用栈示意：

```text
[factorial(1)]
[factorial(2)]
[factorial(3)]
[factorial(4)]
```

等最深层的调用返回后，再一层层出栈并计算结果。

所以：

- **递归太深会占用很多栈空间**
- 如果超过系统分配的栈容量，就会发生**栈溢出（stack overflow）**

---

## 15. 栈溢出（Stack Overflow）

栈溢出指的是：

- 调用栈使用的空间过大
- 超过了系统允许的栈内存范围

常见原因有：

- **无限递归**
- **递归层数过深**
- **局部变量开得太大**

例如下面这种代码就可能导致栈溢出：

```cpp
void f() {
    f();
}
```

因为它会不断调用自己，却永远不返回，调用栈会一直增长。

---

## 16. 调用栈和堆的区别

程序运行时，经常同时提到“栈”和“堆”，但它们不是一回事。

### 栈（Stack）

- 由系统自动管理
- 通常存放函数调用信息、局部变量等
- 申请和释放速度快
- 空间一般较小

### 堆（Heap）

- 通常由程序员或运行时动态管理
- 用于动态内存分配
- 空间通常更大
- 申请和释放相对更复杂

简单理解：

- **调用栈**负责管理函数执行现场
- **堆**负责存放动态创建的数据对象

---

## 17. 为什么调用栈很重要

理解调用栈有助于理解很多程序行为，例如：

- 为什么函数能在执行结束后回到正确位置
- 为什么递归能工作
- 为什么递归过深会报错
- 为什么局部变量在函数结束后会失效
- 为什么调试器里能看到“调用链”

调试程序时看到的：

```text
main -> a -> b -> c
```

本质上就是当前调用栈的展开结果。

---

## 18. 一句话理解调用栈

调用栈就是程序在执行函数时，用来保存**每一层函数调用现场**的栈结构；函数调用时入栈，函数返回时出栈。
