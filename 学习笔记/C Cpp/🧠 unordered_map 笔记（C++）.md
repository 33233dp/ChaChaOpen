
---

# 一、它是什么？

```cpp
unordered_map<int, int> m;
```

👉 **unordered_map 是 C++ STL 里的容器**

### 📌 本质

- ✅ 一个**模板类（template class）**
    
- ✅ 用来存：**键值对（key-value）**
    
- ✅ 底层实现：**哈希表（Hash Table）**
    

---

# 二、它属于哪一类知识？

👉 在 C++ 知识体系中：

```text
STL（标准模板库）
 ├── 容器（containers）
 │     ├── vector（顺序容器）
 │     ├── map（有序）
 │     └── unordered_map（无序）⭐
```

👉 **有序：元素是按规则排好的**  
👉 **无序：元素的排列是乱的，没有固定顺序**

---

# 🧪 先看一个简单例子

## ✅ 有序（ordered）

1, 3, 5, 7, 9

👉 特点：

- 从小到大排好
- 每次看都是这个顺序

---

## ❌ 无序（unordered）

5, 1, 9, 3, 7

👉 特点：

- 没有规律
- 顺序可能变化
---

### 📌 更细分类

👉 **unordered_map 是：**

> ✅ 无序关联容器（unordered associative container）

---

# 三、基本结构

```cpp
unordered_map<key类型, value类型> 变量名;
```

### 🧪 例子

```cpp
unordered_map<string, int> age;
```

👉 表示：

```text
名字 → 年龄
```

---

# 四、核心特点（一定要记）
### ⚡ 1. 查找快
- 平均时间复杂度：**O(1)**
---
### 🔀 2. 无序
- 数据没有固定顺序
- 和 `map` 不一样（map 是有序的）
---
### 🧠 3. 基于哈希表
- 用“哈希函数”定位数据
- 不需要遍历
---
# 五、常用操作

---

## 1️⃣ 插入数据
```cpp
m[key] = value;
```

或：

```cpp
m.emplace(key, value);
//这是插入操作不会覆盖原来已有的。
```

---

## 2️⃣ 查找（重点）

```cpp
m.find(x)
```

### 返回：

- 找到 → 返回位置（迭代器）
    
- 找不到 → 返回 `m.end()`
    

---

### 🔥 判断是否存在

```cpp
if (m.find(x) == m.end())
```

👉 表示：

> ❌ x 不在 map 里

---

## 3️⃣ 访问数据

```cpp
m[key]//❗ key 不存在 → **自动创建一个新元素**
m.at(key);//❌ key 不存在 → **直接报错（异常）**
```

👉 直接拿 value

---

## 4️⃣ 删除

```cpp
//删除其中一个
m.erase(key);
//清除所有
m.clear();
//清除和彻底释放内存
m = unordered_map<char, int>();
```

---

## 5️⃣ 遍历

```cpp
for (auto& p : m) {
    cout << p.first << " " << p.second << endl;
}
```

---

# 六、和 map 的区别（高频考点）

|特性|map|unordered_map|
|---|---|---|
|底层|红黑树|哈希表|
|是否有序|✅ 有序|❌ 无序|
|查找速度|O(log n)|O(1)|
|使用场景|需要排序|只要快|

---

# 七、在算法题中的作用（重点）

👉 主要用途：

### ✅ 1. 快速判断“是否存在”

```cpp
if (m.find(x) != m.end())
```

---

### ✅ 2. 存储“数值 → 下标”

（Two Sum 经典）

---

### ✅ 3. 统计频率

```cpp
m[x]++
```

---

### ✅ 4. 做“补数查找”

```text
target - x
```

---

# 八、典型题目套路（必须掌握）

## 🎯 模型：补数查找

```text
对于每个 x：
    看 x 是否在 map 里
    如果在 → 找到答案
    如果不在 → 存 target - x
```

---

# 九、时间复杂度

|操作|复杂度|
|---|---|
|插入|O(1)|
|查找|O(1)|
|删除|O(1)|

👉 注意：最坏情况 O(n)，但很少发生

---

# 十、本质理解（最重要）

👉 unordered_map 的核心作用：

```text
把“查找某个值”从 O(n) 变成 O(1)
```
